log-analyzer

Fault-tolerant streaming log parser with recovery reporting.

completePython

Overview

A streaming log parser built to keep working when the input doesn't cooperate — malformed lines, truncated files, mixed encodings — rather than halting on the first bad record.

Architecture

Streams the log line-by-line instead of loading it into memory, isolating parse failures per line so one bad record doesn't take down the run.

Engineering decisions

Chose fault isolation over strict parsing: a corrupted line gets logged and skipped rather than aborting the whole pass.

Tradeoffs

Trades strict validation for resilience — malformed input degrades gracefully instead of failing loudly, which is the right call for log data but wouldn't be for something like financial transactions.

Lessons learned

Recovery reporting turned out to matter as much as the parsing itself: knowing *what* was skipped and why is what makes the fault-tolerance trustworthy.
