web-scraper

Fetch/parse split with robots.txt handling and per-host rate limiting.

completePython

Overview

A scraper with a hard separation between fetching and parsing, using requests and BeautifulSoup, with robots.txt compliance and per-host rate limiting.

Architecture

Fetch and parse are separate stages so parsing logic can be tested against fixtures without hitting the network.

Engineering decisions

robots.txt handling fails open — if the file can't be fetched or parsed, the scraper doesn't block itself.

Tradeoffs

Fail-open on robots.txt favors availability over strict compliance in edge cases where the robots file itself is unreachable.

Lessons learned

Ragged tables and malformed HTML were the real long tail of the problem, not the fetch/parse split itself.
