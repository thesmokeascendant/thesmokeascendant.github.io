pdf-parser

Text, table, and metadata extraction with handling for encrypted, corrupted, and scanned PDFs.

completePython

Overview

PDF extraction covering text, tables, and metadata using pypdf and pdfplumber, with explicit handling for encrypted, corrupted, non-PDF, and scanned-page inputs.

Architecture

Layers pypdf for structure/metadata and pdfplumber for text and table extraction, with a reproducible fixture generator for test inputs.

Engineering decisions

Built a fixture generator instead of relying on found PDFs, so edge cases (encrypted, corrupted, scanned) are reproducible in tests.

Tradeoffs

Two extraction libraries instead of one adds dependency weight but covers cases neither handles alone.

Lessons learned

Scanned pages without OCR needed to fail explicitly and legibly rather than silently returning empty text.
