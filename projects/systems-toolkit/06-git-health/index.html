git-health

Repo diagnostics: detached HEAD, stale branches, unpushed commits, large files, conflict markers.

completePython

Overview

A subprocess-based wrapper around the git CLI that checks a repository's working-tree state for common problems before they become incidents.

Architecture

Shells out to git directly rather than reimplementing git internals, parsing CLI output for each check.

Engineering decisions

Checks include detached HEAD, stale branches, unpushed commits, oversized tracked files, and leftover merge-conflict markers.

Tradeoffs

Relying on git's CLI output is more fragile across git versions than a library binding, but far simpler and matches what a human running git would see.

Lessons learned

Tested against real git repos built in temp directories rather than mocked git output, which caught CLI-output parsing bugs early.
