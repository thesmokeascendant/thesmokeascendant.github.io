ssh-automation

Multi-host SSH/SCP runner with transport-vs-command error classification.

completePython

Overview

A subprocess-based multi-host SSH/SCP runner with retry logic and parallel fan-out across hosts, preserving result order.

Architecture

Classifies failures as transport-level (connection, host key) versus command-level (the remote command itself failed) — a real distinction, since only some of those should be retried.

Engineering decisions

Auth and host-key failures are never retried automatically, since retrying those blindly can lock out or mask a real security problem.

Tradeoffs

Parallel fan-out with preserved result ordering adds coordination overhead versus a simple loop, but makes results usable without extra sorting downstream.

Lessons learned

Tested via an injected fake runner, since no real SSH server or network access was available in the build environment.
