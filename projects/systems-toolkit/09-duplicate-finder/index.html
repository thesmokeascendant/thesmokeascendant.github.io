duplicate-finder

Three-stage size/hash funnel — measured 2.79× faster than a naive scan.

completePython

Overview

A duplicate-file finder using a three-stage funnel — file size, then partial hash, then full hash — instead of hashing every file in full up front.

Architecture

Streamed, constant-memory hashing with symlink-loop-safe directory traversal.

Engineering decisions

Measured rather than assumed the speedup: 2.79× faster than a naive full-hash scan on realistic variable-size files, validated at 20,000-file scale.

Tradeoffs

Honestly documents the funnel's worst case too — on uniform-size files, the staged approach is about 10% slower than just hashing everything, since the size-based filter does no work.

Lessons learned

Measuring the actual worst case (uniform-size files) rather than only reporting the best case was what made the benchmark credible.
