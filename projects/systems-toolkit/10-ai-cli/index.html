ai-cli

Pluggable LLM backend with prompt templating and persisted conversation history.

completePython

Overview

A command-line AI client built around an LLMBackend protocol, so the backend (Anthropic or otherwise) is swappable rather than hardcoded.

Architecture

AnthropicBackend implementation uses raw HTTP requests with retry/backoff matching the pattern from project 02 (api-client), plus prompt templating and JSON-persisted, trimmed conversation history.

Engineering decisions

Built the backend as a protocol/interface first, implementation second, so testing didn't depend on network or SDK availability.

Tradeoffs

Raw requests instead of the official SDK means more manual maintenance if the API shape changes, but keeps the dependency footprint minimal.

Lessons learned

Tested via a fake backend and a fake requests.Session, since no network or SDK access was available in the build environment — the protocol-first design is what made that possible.
